## 5. 계산 시트 설계

---

### 📈 [종목현황]

| 컬럼 | 계산 방법 |
|------|-----------|
| 계좌 / 유형 / 통화 | 거래내역 고유값 |
| 종목코드 / 종목명 | 거래내역 참조 |
| 보유수량 | 매수합계 − 매도합계 (SUMPRODUCT) |
| 평균매입가 | 총매입금 ÷ 총매입수량 |
| **현재가** | `GOOGLEFINANCE(코드)` 자동 |
| 평가금 (원래통화) | 수량 × 현재가 |
| 투자금 (원래통화) | 수량 × 평균매입가 |
| **평가금(₩)** | USD: 평가금 × 실시간환율 / KRW: 그대로 |
| **투자금(₩)** | USD: 거래환율 기반 / KRW: 그대로 |
| 수익금(₩) | 평가금(₩) − 투자금(₩) |
| **수익률(%)** | 수익금 ÷ 투자금 × 100 |
| 투자비중(%) | 종목평가금 ÷ 전체평가금 |
| 배당누계(₩) | 배당내역 SUMIF |

> ⚠️ 보유수량 = 0 (전량 매도) 종목은 자동 필터링

---

### 🏦 [계좌현황]

| 컬럼 | 계산 방법 |
|------|-----------|
| 계좌 / 증권사 / 유형 | 설정시트 참조 |
| 종목수 | 해당 계좌 보유 종목 수 |
| 투자금(₩) | 종목현황 SUMIF (원화 통합) |
| 평가금(₩) | 종목현황 SUMIF (원화 통합) |
| 수익금(₩) | 평가금 − 투자금 |
| **수익률(%)** | **수익금 ÷ 투자금 × 100** |
| 입금누계(₩) | 입출금내역 SUMIF |
| 배당누계(₩) | 배당내역 SUMIF (원화환산 합계) |
| 비중(%) | 계좌평가금 ÷ 전체평가금 |

---

### 📉 [월별현황]

| 컬럼 | 계산 방법 |
|------|-----------|
| 년월 | 월말스냅샷 고유값 |
| 총 평가금(₩) | 전 계좌 원화환산 합산 |
| 누적 입금(₩) | 해당 월까지 입출금 누계 |
| 월 수익(₩) | 이번달 평가금 − 지난달 평가금 − 이번달 신규입금 |
| 누적 수익률(%) | 전체수익 ÷ 누적투입금 × 100 |

**자동 생성 차트 2개**
- 📊 누적 자금 흐름 (면적): 누적입금 vs 총평가금
- 📊 월별 수익 (막대): 수익=초록, 손실=빨강

---

### 🎯 [대시보드]

**요약 카드 (6개)**
- 총 평가금(₩) / 총 투자금(₩) / 총 수익금+수익률 / 배당누계(₩) / 누적입금(₩) / 실시간 환율

**차트** — 월별현황 데이터 참조

**계좌별 요약 테이블** — 계좌현황 참조

**종목별 TOP5 / BOTTOM5** — 수익률 기준

## 6. 환율 처리 정리

| 용도 | 환율 | 방식 |
|------|------|------|
| 현재 평가금 원화 환산 | 실시간 | `GOOGLEFINANCE("CURRENCY:USDKRW")` 자동 |
| 거래내역 매입단가 기록 | 거래 당시 환율 | 기본값 자동 채움, 필요 시 수정 |
| 입출금 달러환산 | 입금 당시 환율 | 기본값 자동 채움, 필요 시 수정 |
| KRW 계좌 | 해당 없음 | 환율=1 고정 |

## 8. 개발 순서 (예정)

```
1단계  ⚙ 설정 시트 제작 (계좌 목록 + 실시간 환율)

```
